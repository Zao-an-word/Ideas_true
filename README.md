## 📦 项目简介
“文思成真”是面向小说创作者、编剧、插画师等群体打造的**AI 辅助创作全流程平台**，针对现有创作工具风格单一、多模态协作能力不足、创作流程割裂等痛点，基于**FastAPI + Vue 3** 技术栈构建，深度整合大语言模型（Deepseek/Qwen/OpenAI）、多模态生成（通义万相）与向量检索（RAG）技术，提供从小说智能生成、文风适配、参考文献管理，到图文可视化、人物关系图/地图自动生成的一站式创作支持。

平台采用**多智能体工具链**与**异步处理架构**，保障创作效率与内容质量，同时支持用户管理、作品库沉淀、数据加密等平台化能力，适配个人创作、教学练习、团队协作等多种使用场景，助力创作者缩短创作周期、突破创作瓶颈。

## 🎯 核心功能
### 🔍 小说生成引擎
- **智能生成**：基于关键词/人设/情节方向，结合参考文献生成逻辑连贯的小说文本，支持武侠、言情等多风格适配，单轮生成≥200字
- **续写与润色**：基于对话历史保持人设与情节一致性，自动语法优化、风格统一，润色后与前文风格差异率≤10%
- **多模型切换**：无缝切换Qwen/Deepseek/OpenAI等模型，支持自定义生成随机性（temperature参数）

### 📚 参考文献管理（RAG架构）
- **多格式上传**：支持PDF/Word/TXT格式，单文件≤10MB，批量上传≤5个/30MB，自动校验格式与大小
- **智能解析**：自动提取正文、清洗冗余内容，排除页眉/页脚/水印等非有效信息
- **语义检索**：将参考资料向量化存入Chroma，创作时精准检索高相关度内容，提升生成内容的风格一致性与知识准确度

### 🎨 多媒体生成与可视化
- **文本生图**：根据场景描述生成多风格图片（写实/卡通/油画），支持高清分辨率（最高2048×2048），图文相似度≥75%
- **人物关系图**：从小说文本自动提取核心人物及关系，生成Mermaid语法可视化图谱，支持SVG/PNG格式导出
- **地图生成**：解析地理描述提取地形/方位信息，生成无文字基础地图并智能标注，实现小说世界观可视化

### ✏️ 在线文本编辑
- **富文本编辑**：基于TinyMCE实现多级标题、字体样式、拼写检查，支持小说内容在线修改与格式调整
- **版本管理**：自动保存创作历史版本，支持版本对比与回溯，编辑内容可一键上传至个人作品库

### 👤 平台化管理功能
- **用户管理**：支持注册/登录、个人资料编辑（头像/昵称）、密码修改，基于JWT实现安全认证
- **作品库沉淀**：创作内容自动关联个人作品库，支持按创作时间/类型/标签分类检索，可将作品共享为参考资料
- **管理员功能**：用户列表管理、违规内容审核、创作数据统计，基于RBAC实现角色权限控制
- **数据安全**：用户数据加密存储、HTTPS传输，支持个人数据导出与账号注销，平台级AI合规校验拦截违规内容

### ⚡ 性能与体验优化
- **异步生成**：文本/图片生成为异步模式，前端显示加载状态，1000字文本生成≤30秒，单张图片生成≤60秒
- **高并发支持**：后端异步接口+多进程部署，前端缓存机制，支持并发用户≥100，高峰期通过RabbitMQ队列调度
- **极简操作**：核心创作功能操作步骤≤3步，关键按钮（生成/保存）尺寸≥48×48px，创作工作台直观易用

## 🚀 快速部署
> 部署前请确保已满足【环境要求】，并已克隆本项目至本地：
> ```bash
> git clone <项目仓库地址>
> cd 文思成真
> ```
> 详细部署步骤可参考项目内**安装部署说明书**，包含分步操作与细节注意事项。

### ① 数据库初始化
#### 步骤1：安装MySQL
- Windows：使用MySQL Installer一键安装，配置环境变量与服务端口
- Mac：`brew install mysql`
- Linux：`apt install mysql-server` / `yum install mysql`

#### 步骤2：初始化数据库与表结构（二选一）
##### 方式1：自动建表（推荐）
```bash
python Backend/init_db.py
```
##### 方式2：SQL文件导入
```bash
# 先创建空数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS novel_db DEFAULT CHARACTER SET utf8mb4;"
# 导入表结构
mysql -u root -p novel_db < text_generator.sql
```

### ② 后端部署（FastAPI）
#### 步骤1：创建并激活虚拟环境
##### 方式1：使用environment.yml（推荐，自动安装依赖）
```bash
conda env create -f environment.yml
conda activate novel_env
```
##### 方式2：手动创建
```bash
# 创建Conda虚拟环境
conda create -n novel_env python=3.12 -y
conda activate novel_env
# 进入后端目录并安装依赖
cd Backend
pip install --upgrade pip
pip install -r requirements.txt
```

#### 步骤2：配置数据库与安全参数
编辑 `Backend/app/core/config.py` 文件，修改以下核心配置：
```python
# 数据库配置
DB_HOST = "localhost"       # 数据库地址
DB_PORT = 3306              # 数据库端口
DB_USER = "root"            # 数据库用户名
DB_PASSWORD = "yourpassword"# 数据库密码（替换为实际密码）
DB_NAME = "novel_db"        # 数据库名
DB_CHARSET = "utf8mb4"      # 字符集，支持emoji与特殊字符

# 安全配置
SECRET_KEY = "your_secret_key"  # 替换为随机生成的密钥（建议≥32位）
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 30  # JWT Token过期时间

# AI模型配置（按需填写）
DASHSCOPE_API_KEY = "your_dashscope_key"
OPENAI_API_KEY = "your_openai_key"
```

#### 步骤3：启动后端服务
```bash
cd Backend
# 开发环境（热重载，适合开发调试）
uvicorn main:app --reload --host 0.0.0.0 --port 8000
# 生产环境（禁用热重载，多进程部署，适合线上运行）
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

#### 步骤4：验证后端启动
访问接口文档地址，确认所有接口正常加载：
- Swagger UI：[http://localhost:8000/docs](http://localhost:8000/docs)

### ④ 环境变量配置（可选，推荐）
为避免硬编码敏感信息（如数据库密码、AI模型API密钥），推荐使用`.env`文件管理环境变量，步骤如下：
1. 在`Backend`目录下创建`.env`文件：
```env

DB_PORT=3306

# AI模型API配置
DASHSCOPE_API_KEY=your_dashscope_key

DB_PASSWORD = os.getenv("DB_PASSWORD")
DB_NAME = os.getenv("DB_NAME", "novel_db")
SECRET_KEY = os.getenv("SECRET_KEY")
# 其他配置同理
```

## 🧱 数据结构说明
项目采用MySQL关系型数据库，严格遵循**三范式**设计，核心表结构包含用户、作品、会话、消息等，支持级联删除、索引优化，保障数据查询与操作效率，**完整表结构与SQL语句**详见`text_generator.sql`与`Backend/app/db/init.sql`。

### 核心数据表概览
| 表名         | 核心用途 | 关键索引/约束 |
|--------------|----------|---------------|
| `user`       | 存储用户基本信息、认证信息、个人资料 | 主键：id；唯一索引：username/phone；外键：无 |
| `novels`     | 存储用户创作的小说信息（标题/内容/封面等） | 主键：id；普通索引：user_id；外键：user_id（关联user） |
| `chapters`   | 存储小说章节信息，按小说关联与章节号排序 | 主键：id；普通索引：novel_id；外键：novel_id（关联novels） |
| `conversations` | 存储用户与AI的创作会话记录 | 主键：id；普通索引：user_id/novel_id；外键：user_id（关联user） |
| `messages`   | 存储会话中的具体消息（用户输入/AI输出） | 主键：id；普通索引：conversation_id；外键：conversation_id（关联conversations） |
| `collections` | 存储用户的小说收藏记录，避免重复收藏 | 主键：id；唯一索引：user_novel；外键：user_id/novel_id |

### 数据流转规则
- 小说生成：用户输入→结构化解析→临时表存储→AI模型生成→结果存入novels/chapters→删除临时表数据
- 会话交互：用户消息→存入messages→关联conversation→AI模型响应→结果存入messages→更新conversation
- 参考文献：文件上传→解析清洗→向量化→存入Chroma向量库→创作时语义检索→生成内容关联参考资料

## 📖 使用指南
### 首次使用流程
1. **环境验证**：确认前后端服务均已启动，访问[http://localhost:8080](http://localhost:8080)进入前端页面
2. **用户注册/登录**：点击页面右上角「注册」，填写用户名/密码/手机号完成注册，或使用测试账号登录（默认管理员账号：admin / 123456，建议首次登录后修改密码）
3. **创作准备**：进入「个人中心」，可编辑个人资料、上传头像，提前准备创作所需的参考文献（支持PDF/Word/TXT）
4. **开始创作**：进入「创作工作台」，选择创作类型，开始小说生成与编辑

### 核心创作操作
#### 小说生成
1. 进入「小说创作」模块，输入**关键词/人设/情节方向**，可选择上传参考文献（可选）
2. 选择生成模型与创作风格，点击「生成」，等待异步生成结果（前端显示“生成中”）
3. 生成完成后，可对文本进行**续写/润色**，或直接在富文本编辑器中修改编辑
4. 编辑完成后，点击「保存」，内容自动存入个人作品库，支持添加标签与分类

#### 多媒体可视化
1. **文本生图**：在小说编辑页，选中需要配图的场景描述，点击「生成配图」，选择图片风格，等待生成结果，支持高清下载
2. **人物关系图**：进入小说的「可视化」模块，点击「生成人物关系图」，系统自动提取人物及关系，生成可视化图谱，支持SVG/PNG导出
3. **地图生成**：输入小说世界观的地理描述，点击「生成地图」，系统自动解析并生成标注好的地图，支持自定义地图尺寸与风格

#### 作品管理
1. 进入「个人作品库」，可查看所有创作的小说，支持按**创作时间/类型/标签**检索
2. 对小说进行**编辑/删除/分享**，分享后的作品可作为其他用户的参考文献
3. 点击小说标题，可查看小说详情、章节列表，支持继续创作与导出

### 管理员操作
1. 使用管理员账号登录，进入「后台管理」模块
2. **用户管理**：查看所有注册用户列表，支持禁用违规用户、导出用户统计数据
3. **内容审核**：查看用户上传的参考文献与生成的小说内容，拦截/删除违规内容并通知用户
4. **数据统计**：查看平台整体创作数据（累计生成字数/作品数/活跃用户数）

|------------------|-------------------------------------------|--------------------------------------------------------------------------|
| 数据库连接失败    | 配置错误/MySQL未启动/端口被占用/权限不足 | 1. 检查config.py/.env中数据库参数<br>2. 启动MySQL服务，检查3306端口<br>3. 赋予MySQL用户远程访问权限 |
| 后端启动报错      | 虚拟环境未激活/依赖未安装/端口被占用 | 1. 执行`conda activate novel_env`激活环境<br>2. 重新执行`pip install -r requirements.txt`<br>3. 更换端口（如--port 8001） |
| 前端页面空白      | 后端未启动/接口地址错误/跨域配置错误 | 1. 确认后端服务正常运行<br>2. 检查.env.development中VUE_APP_API_BASE_URL<br>3. 确认后端CORSMiddleware包含前端地址 |
| 依赖安装失败      | Python/Node版本不匹配/网络问题/镜像源错误 | 1. 检查版本是否满足要求（Python≥3.12，Node≥18.x）<br>2. 切换国内镜像源<br>3. 升级pip/npm至最新版 |

### AI功能调用问题
| 问题             | 可能原因                                  | 解决建议                                                                 |
|------------------|-------------------------------------------|--------------------------------------------------------------------------|
| 小说生成无响应    | API密钥错误/网络无法访问公网/模型调用超限 | 1. 检查config.py/.env中AI模型API密钥<br>2. 验证公网访问能力<br>3. 检查模型平台调用额度 |
| 图文生成失败      | 文本描述过短/图文相似度不达标/模型服务异常 | 1. 补充详细的场景/地理描述<br>2. 重新点击生成，系统自动重试<br>3. 检查DashScope服务状态 |
| 参考文献检索无结果 | 文件未上传成功/格式不支持/内容未解析 | 1. 确认文件上传成功（单文件≤10MB）<br>2. 仅支持PDF/Word/TXT格式<br>3. 重新上传文件，系统自动重新解析 |

### 接口与安全问题
| 问题             | 可能原因                                  | 解决建议                                                                 |
|------------------|-------------------------------------------|--------------------------------------------------------------------------|
| 跨域错误（CORS） | 后端未配置允许的前端源地址                | 修改后端main.py中CORSMiddleware：<br>`allow_origins = ["http://localhost:8080"]` |
| 401/403认证错误  | Token过期/未携带Token/权限不足            | 1. 重新登录获取Token<br>2. 确认请求头携带`Authorization: Bearer <token>`<br>3. 检查用户角色与接口权限 |
| 400/422参数错误  | 参数缺失/类型不匹配/格式错误              | 1. 检查前端请求参数是否完整<br>2. 确保参数类型与后端接口一致<br>3. 参考Swagger接口文档调整参数 |
| 500服务器内部错误 | 代码逻辑bug/数据库查询错误/空指针异常     | 1. 查看后端运行日志，定位错误代码<br>2. 检查数据库表/字段是否存在<br>3. 验证输入参数是否为有效值 |
| 413文件过大错误  | 上传文件超过限制                          | 1. 确认文件大小符合要求（单文件≤10MB）<br>2. 前端上传前增加文件大小校验<br>3. 拆分大文件后分批上传 |

### 性能问题
| 问题             | 可能原因                                  | 解决建议                                                                 |
|------------------|-------------------------------------------|--------------------------------------------------------------------------|
| 生成速度过慢      | 硬件配置不足/并发量过高/模型参数设置不当 | 1. 升级硬件（≥16G内存）<br>2. 高峰期减少并发请求<br>3. 适当降低模型生成参数（如max_tokens） |
| 页面操作卡顿      | 浏览器缓存过多/前端资源未加载完成        | 1. 清理浏览器缓存与Cookie<br>2. 刷新页面等待资源加载完成<br>3. 关闭浏览器多余标签页 |

## 🤝 贡献指南
本项目欢迎开发者提交Issue和Pull Request参与迭代开发，协作遵循以下规范：
1. **分支管理**：主分支`main`（仅存储稳定可运行代码），功能开发基于`feature/xxx`分支，Bug修复基于`fix/xxx`分支
2. **提交规范**：提交信息遵循`类型: 描述`格式，如`feat: 新增小说标签检索功能`、`fix: 修复数据库连接超时问题`
3. **开发规范**：
   - 后端：遵循PEP8编码规范，接口添加注释，基于Pydantic做参数校验
   - 前端：遵循Vue官方开发规范，组件化开发，避免冗余代码
4. **PR提交**：提交PR前需确保代码本地测试通过，无语法错误与功能Bug，附带详细的修改说明
5. **问题反馈**：提交Issue时需注明**环境版本、问题描述、复现步骤、报错日志**，便于快速定位解决

> 团队内部协作遵循统一的开发流程与文档规范，所有代码修改需同步更新相关设计与实现文档。
